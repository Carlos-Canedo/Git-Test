Git Test

Basic commands

  • git init: create a new git project or start a project using git.
  • add +file: to pass files from working directory to staging area.
  • git status: used to see where are files (directory, staging area).
  • git commit: to pass files from staging area to repository (create a snapshoot of code version).
  • git push: to upload to a remote repository.
  • git pull: get new changes from other users
  • git clone: makes a copy from central version to a personal computer.

Other commands

  • ls: command to list directories and files.
  • pwd: helps to see which path we are.
  • clear: clear console

Create repository

  • Right click in the project folder.
  • Select “Git Bash Here” (a terminal will open)
  • With “ls” we can see all the files on the folder selected.
  • Put “git init” in the bash to initialize empty fit repository in the folder (it will create a hidden folder in which it will be saved the local repository).
  • “git status” to see the status of our files.
  • “git add +file_name” to add a file to status area.
  • Config email with “git config –global user.email “my_email@example.com””.
  • Config name with “git config –global user.name “my_name””.
  • “git commit” to create a start point in our repository (it will open a code editor).
  • Press “i” key to start to write on the editor, write the commit title.
  • Press scape key and write “:wq” to get back.
  • “git log” to watch all the commits made.
  • “git status” to see for some modify file.

Discard changes until the last commit (only works if the file wasn't added to repository)

  • “git checkout -- +file_name”.

Amend commit

  • “git commit –amend” and you will acces to the actual commit.

See all changes of a file from the commit

  • “git diff +file_name”.

Add changes of a file

  • “git add +file_name”.

Make a new commit

  • “git commit”, press I key and write the commit title, esc key and write “:wq” to get back.
  • Option two: “git commit -m “commit_title””.

Ignore files from our repository

  • Create a file in the folder main repository with the name “.gitignore”.
  • Write in all the files you want to ignore from the repository.
  • Add “.gitignore” to repository (“git add .gitignore“).
  • Optionally make a commit for the changes (“git commit”, press I key and write the commit title, esc key and write “:wq” to get back.).

Create Branch (default “master”)

  • Consult branch “git branch”.
  • Create branch “git branch branch_name”.
  • Get into branch: “git checkout “branch_name””.

Add all files to repository

  • “git add .”.

Create a git-hub repository

  • Login github.
  • Open repositories.
  • Select new.
  • Give a name to repository.
  • Click on create repository button.
  • Put the remote instruction made by git-hub in the batch of your local repository (example: git remote add origin https://github.com/Carlos-Canedo/Git-Test.git).
  • Upload to remote repository: git push -u origin master (master means the batch which was added the remote origin).

Clone repository

  • Go to the folder in which you want to clone the repository.
  • Open the GitBatch console.
  • Put the next instruction: “git clone +url_Git-hub-repository” (example: git clone https://github.com/Carlos-Canedo/Git-Test.git).
